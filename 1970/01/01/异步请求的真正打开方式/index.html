# Promise的方式

getKaptcha() {
    getKaptcha().then((res) => {
    this.imgAuth.imgCode = res.check_code_img;
    this.imgAuth.tokencode = res.check_code_token;
    });
},

getKaptcha是一个axios请求，返回的是个Promise。 👆的代码是Promise的写法，使用then指定回调函数。

# 👇是异步请求的真正打开方式

async getKaptcha() {
    const res = await getKaptcha();
    this.imgAuth.imgCode = res.check_code_img;
    this.imgAuth.tokencode = res.check_code_token;
},

是不是很“同步”啊！？，看起来getKaptcha() 这个axios异步请求就像是同步一样。

为啥可以用类似同步的方式去写异步呢？奥秘就在await上，实际上await就是个then的语法糖。用一个变量去接受await的返回值，返回的就是本来由then接受的resolve值。

# 那么Promise就被async完全替代了吗？

非也。如果想实现并联异步请求还是要用Promise.all或者Promise.race去实现

let p1 = promiseRequest1();
let p2 = promiseRequest2();
let [res1, res2] = await Promise.all([p1, p2]);
console.log(res1)
console.log(res2)

# 那么await会阻塞线程吗？

不会,await只是then的语法糖

当运行到await时候，就会会发起异步请求，推入异步任务队列，然后继续执行其他的同步代码。然后异步结束后，再调用await的赋值。

# async/await的错误处理

上面说的都是异步方法运行成功的例子，如果异步方法报错该怎么办。

  • 使用try/catch捕获异常
async getKaptcha() {
    try{
        const res = await getKaptcha();
    }catch(error){
        console.log(error)
    }
    this.imgAuth.imgCode = res.check_code_img;
    this.imgAuth.tokencode = res.check_code_token;
},
  • 使用.catch()进行异常捕获
async getKaptcha() {
    const res = await getKaptcha()
        .catch(error=>{
            console.log(error)
        });
    this.imgAuth.imgCode = res.check_code_img;
    this.imgAuth.tokencode = res.check_code_token;
},
