# 防抖

先看代码

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width,initial-scale=1.0,maximum-scale=1.0,user-scalable=no">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <title>防抖</title>
</head>
<body>
  <button id="debounce">点我防抖！</button>

  <script>
    window.onload = function() {
      // 1、获取这个按钮，并绑定事件
      var myDebounce = document.getElementById("debounce");
      myDebounce.addEventListener("click", debounce(sayDebounce));
    }

    // 2、防抖功能函数，接受传参
    function debounce(fn) {
      // 4、创建一个标记用来存放定时器的返回值
      let timeout = null;
      return function() {
        // 5、每次当用户点击/输入的时候，把前一个定时器清除
        clearTimeout(timeout);
        // 6、然后创建一个新的 setTimeout，
        // 这样就能保证点击按钮后的 interval 间隔内
        // 如果用户还点击了的话，就不会执行 fn 函数
        timeout = setTimeout(() => {
          fn.call(this, arguments);
        }, 1000);
      };
    }

    // 3、需要进行防抖的事件处理
    function sayDebounce() {
      // ... 有些需要防抖的工作，在这里执行
      console.log("触发事件处理函数");
    }

  </script>
</body>
</html>

这里利用了定时器的方法来实现防抖
这个🌰中，利用了闭包的原理，共用了同一个变量timeout,在之后的每次触发click事件后，都清除了这个timeout，这样就不会触发事件处理函数。直至1000ms内没有触发防抖函数，也就是没有清除定时器，才执行事件处理函数。
具体表现为：如果一直点击按钮不停，那么就不会触发事件，直至两次触发的时间间隔大于设定的1000ms，才会触发一次事件处理函数。

# 节流

先看代码

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width,initial-scale=1.0,maximum-scale=1.0,user-scalable=no">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <title>节流</title>
</head>
<body>

  <button id="throttle">点我节流！</button>

  <script>
    window.onload = function() {
      // 1、获取按钮，绑定点击事件
      var myThrottle = document.getElementById("throttle");
      myThrottle.addEventListener("click", throttle(sayThrottle));
    }

    // 2、节流函数体
    function throttle(fn) {
      // 4、通过闭包保存一个标记
      let canRun = true;
      return function() {
        // 5、在函数开头判断标志是否为 true，不为 true 则中断函数
        if(!canRun) {
          return;
        }
        // 6、将 canRun 设置为 false，防止执行之前再被执行
        canRun = false;
        // 7、定时器
        setTimeout( () => {
          fn.call(this, arguments);
          // 8、执行完事件（比如调用完接口）之后，重新将这个标志设置为 true
          canRun = true;
        }, 1000);
      };
    }

    // 3、需要节流的事件
    function sayThrottle() {
      console.log("触发事件处理函数");
    }

  </script>
</body>
</html>

这里也是利用了定时器（中间夹杂着闭包）来实现节流
给click事件添加一个监听事件：这个监听事件共用一个闭包量canRun，当第一次触发click事件时，将canrun设置成false,然后再设置个定时器：再1000ms后，触发事件处理函数，并把canrun设置成true。由于在闭包函数中，一开始就设置个关卡：如果canrun为false就不执行下去。

就导致了一个现象：事件处理函数，只会在点击后，1000ms后才触发。并且1000ms内也只会触发一次。

# 相同点

两者的事件处理函数，只会在事件触发后，大于等于规定的时间后（1000ms）才会触发。

# 区别

防抖：在最后一次事件触发后的，超过规定时间（1000ms）才会触发事件处理函数。
节流：第一次事件触发后，超过规定时间（1000ms）才会触发事件处理函数。1000ms内的触发全部忽略，触发完事件处理函数后，重新监听第一次事件触发。

# 比喻

防抖：电脑的自动睡眠功能，你一直使用电脑（触发事件），那么电脑不会自动睡眠（触发事件处理函数），如果你一段时间（比如5分钟）不用电脑，那么电脑就自动睡眠（触发事件处理函数）

节流：电脑的强制休息功能，（每小时提醒你自动站起来活动一下）。从你用电脑开始计算（触发事件），1h后就会提醒你运动一下（触发事件处理函）。
