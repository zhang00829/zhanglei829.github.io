# Promise 基本用法

基本用法，网上一搜一大把，在此就不赘述了。

说一个注意点，方便大家理解Promise

Promise在新建后就会立即执行

let promise = new Promise(function(resolve, reject) {
  console.log('Promise');
  resolve();
});

promise.then(function() {
  console.log('resolved.');
});

console.log('Hi!');

// Promise 
// Hi!
// resolved

在第一行创建一个Promise对象后，内部的方法就已经执行了。如果方法是一个异步的ajax请求，则会在Promise创建时候就已经发送出请求了。
然后，在异步请求结束后（ajax请求回来后），才调用then后面的方法。
其实，promise的then方法就是给Promise中的异步方法注册回掉函数的。

# Promise封装ajax

const getJSON = function(url) {
  const promise = new Promise(function(resolve, reject){
    const handler = function() {
      if (this.readyState !== 4) {
        return;
      }
      if (this.status === 200) {
        resolve(this.response);
      } else {
        reject(new Error(this.statusText));
      }
    };
    const client = new XMLHttpRequest();
    client.open("GET", url);
    client.onreadystatechange = handler;
    client.responseType = "json";
    client.setRequestHeader("Accept", "application/json");
    client.send();

  });

  return promise;
};

getJSON("/posts.json").then(function(json) {
  console.log('Contents: ' + json);
}, function(error) {
  console.error('出错了', error);
});

# Promise链式调用

Promise的then方法，返回的也是个Promise对象，所以可以进行链式调用

getJSON("/posts.json").then(function(json) {
  return json.post;
}).then(function(post) {
  // ...
});

👆then里面return一个josn.post，then会把它包装成一个新的Promise对象。
如果第一个then中返回的是另外一个异步方法，那么就实现形成串联调用后台接口了

axios是常用的请求后台接口的库，很强大。axios的请求也是一个Promise对象。

axios.get('https://xxx/api/xxx').then(res=>{
     //do something
    return axios.get('https://xxx/api/xxx')
}).then(res=>{
    //do something
}).catch(err=>{
    console.log('error: ',err)
})

👆的代码才是串联请求的真正打开方式，而不是👇

axios.get('https://xxx/api/xxx').then(res=>{
     //do something
     axios.get('https://xxx/api/xxx').then(response=>{
         //do something
     })
})

这种写法还是没能跳开回掉的思维方式，实际上还是会产生回调地狱。

# 异常捕获

Promise中异步方法可能成功，也可能失败。成功时候调用resolve（'xxx'），返回成功的信息xxx；如果失败调用reject（‘xxx’）返回错误信息xxx。

Promise的异常捕获有2种，

  • 一种是then方法内，传入两个方法参数，第一个方法参数会处理成功的方法，第二个方法参数会处理失败的方法。
  • 第二种是使用Promise的catch方法。
p.then((val) => console.log('fulfilled:', val),(err) => console.log("rejected:", err))

// 等同于
p.then((val) => console.log('fulfilled:', val))
  .catch((err) => console.log('rejected', err));

常用的是第二种方法，因为如果存在链式调用，则只需要一个catch全部捕捉前面的异常。

# Promise.all()

这是个很有用的方法，可是实现并行请求。

const promise1 = axios.get('xxx');
const promise2 = axios.get('xxx');
const promise3 = axios.get('xxx');

const p = Promise.all([promise1,promise2,promise3])

p.then(([p1,p2,p3])=>{ //这里使用了结构赋值
  //处理 接口1，接口2，接口3返回的数据
})

p.then的处理函数只会在接口1，接口2，接口3，都完成时，才会调用。

# Promise.race()

const promise1 = axios.get('xxx');
const promise2 = axios.get('xxx');
const promise3 = axios.get('xxx');

const p = Promise.reace([promise1,promise2,promise3])

p.then(([p1,p2,p3])=>{ //这里使用了结构赋值
  //处理 接口1，接口2，接口3返回的数据
})

也是可以并行调用接口，但是与Promise.all()不同的是，只要有一个完成就调用p.then内的方法
